# Zion Franchise Character Standard

## Christian Surfers™ Permanent Character Asset Pipeline Standard

This document defines the mandatory approval system for Zion, the franchise benchmark character for Christian Surfers™.

Zion is not only a playable character. Zion is the face of the brand. Every future Christian Surfers character must be judged against the standard Zion establishes.

No final roster character may enter production until Zion passes every gate in this document.

---

# Gate 0 — Franchise Test

Before Zion enters the approval ladder, answer one question:

> If Christian Surfers becomes a Top 100 mobile game and 1 million children see this character, is this the version of Zion we want representing the brand for the next 10 years?

If the answer is any of the following:

```txt
Maybe.
Good enough.
We can improve him later.
He'll do for now.
Close enough.
This is fine for launch.
We can replace him in version 2.
```

Then Zion fails.

```txt
FAIL
```

Only proceed if the answer is:

> Yes. This is the Zion we want on App Store screenshots, trailers, merchandise, loading screens, YouTube thumbnails, promotional art, and future sequels.

If that answer cannot be said confidently, Zion fails.

---

# Gate 1 — Identity Approval

Can a player understand who Zion is emotionally?

Zion must clearly communicate:

- hopeful
- brave
- purpose-driven
- Kingdom runner

The player should emotionally understand Zion before reading a name, bio, ability, or scripture.

If Zion does not communicate this identity:

```txt
FAIL
```

---

# Gate 2 — Silhouette Approval

Can Zion be recognized by silhouette alone?

Zion must not read as:

- a runner
- a kid
- a game character
- a blue hoodie
- a generic mobile avatar

Zion must read as:

```txt
Zion
```

Required silhouette checks:

- standing pose
- back-facing run pose
- back-left run pose
- back-right run pose
- jump pose
- slide pose
- victory pose

If Zion is not recognizable by silhouette alone:

```txt
FAIL
```

---

# Gate 3 — Back-View Approval

Can Zion be identified from behind?

This is mandatory because Christian Surfers gameplay is primarily:

- third-person chase camera
- back-facing runner
- slightly elevated perspective
- movement toward golden light / horizon

Zion must remain identifiable from:

- direct back view
- back-left view
- back-right view
- running away from the camera
- jumping away from the camera
- sliding away from the camera

If Zion cannot be identified from behind:

```txt
FAIL
```

---

Can Zion be recognized at actual gameplay scale?

```txt
96x96
128x128
```

- hair shape
- jacket silhouette
- color blocking
- shoe shape
- back identity

- movement attitude

```txt
FAIL
```

If Zion becomes only a generic blue hoodie or unreadable runner at thumbnail scale:

```txt
FAIL
```

# Gate 5.5 — Recognition Test

Zion must pass recognition testing before approval.

## 1-Second Test

Show Zion's silhouette for exactly:

```txt
1 second
```

Ask:

```txt
Who is this?
```

Required answer:

```txt
That's Zion.
```

Acceptable answers:

```txt
Zion.
The Christian Surfers main character, Zion.
```

Failing answers:

```txt
That's a runner.
That's a kid.
That's a game character.
That's a blue hoodie.
The main guy.
I'm not sure.
```

If testers identify the category but not the character:

```txt
FAIL
```

```txt
FAIL
```

# Gate 7 — Technical Approval

Zion final assets must pass every technical requirement:

```txt
transparent PNG
clean alpha
no background
no text
no UI
no card frame
consistent canvas
consistent scale

contact sheet created
```

- no contact sheet used as live gameplay art

- consistent character height across frames
- transparent RGBA export verified

If the asset is technically dirty:

```txt
FAIL
```

# Christian Surfers Character Law

## 1. No Character May Exceed Zion

No future roster character should look more iconic, more polished, more memorable, or more franchise-defining than Zion.

Other characters can be excellent, but Zion must remain the visual anchor.

If another character makes Zion feel secondary:

```txt
FAIL
```

## 2. No Character May Ignore Zion

Every future character must respect the visual world Zion establishes:

- proportions
- lighting
- rendering quality
- silhouette exaggeration
- outfit complexity
- shoe size
- glow intensity
- emotional tone
- mobile readability
- Christian Surfers identity

If a character feels disconnected from Zion's design language:

```txt
FAIL
```

## 3. No Character May Redefine The Visual Style After Zion Is Approved

Once Zion passes Gate 0 through Gate 7, his approved design becomes the franchise style bible.

Future characters may vary by:

- color
- hairstyle
- outfit silhouette
- personality
- rarity
- power-up flavor
- movement style

But they may not redefine:

- rendering style
- proportion system
- visual tone
- camera readability
- Christian Surfers identity
- premium quality bar

If a character looks like they came from another game:

```txt
FAIL
```

## 4. Every Future Character Must Belong In Zion's World

Every roster character must feel like they could stand next to Zion in:

```txt
App Store screenshots
trailers
loading screens
merchandise
YouTube thumbnails
promotional art
future sequels
```

And the viewer should think:

```txt
These characters belong to the same world.
```

If a character weakens Zion's identity:

```txt
FAIL
```

If a character strengthens Zion's world:

```txt
PASS
```
